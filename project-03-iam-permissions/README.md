# Project 3: IAM Permissions and Security

## Overview

This project demonstrates how to create and manage AWS Identity and Access Management (IAM) users, groups, and permissions following security best practices.

## Objectives

- Create IAM users
- Create IAM groups
- Assign permissions using policies
- Implement least-privilege access
- Improve account security

## AWS Services Used

- AWS Identity and Access Management (IAM)

## Skills Demonstrated

- Identity and Access Management
- Security Best Practices
- User Administration
- Group Management
- Policy Configuration
- Least-Privilege Access Control

## Steps Performed

1. Created IAM users
2. Created IAM groups
3. Attached permission policies
4. Tested user access
5. Verified permissions and restrictions

## Outcome

Successfully configured IAM users and groups with appropriate permissions while following AWS security best practices.

## Lessons Learned

- The difference between users, groups, and policies
- Why least-privilege access is important
- How IAM improves AWS account security
- How to manage permissions safely and effectively
