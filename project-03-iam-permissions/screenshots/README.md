# Project 3: IAM Permissions and Security

## Overview

This project demonstrates how to create and manage AWS Identity and Access Management (IAM) users, groups, and permissions following security best practices.

## Objectives

- Create IAM users
- Create IAM groups
- Assign permissions using policies
- Implement least-privilege access
- Improve account security

## AWS Services Used

- AWS IAM

## Skills Demonstrated

- Identity Management
- Access Control
- Security Best Practices
- Policy Management
- User Administration

## Steps Performed

1. Created IAM users
2. Created IAM groups
3. Attached permission policies
4. Tested user permissions
5. Verified access restrictions

## Outcome

Successfully implemented role-based access controls and user permissions within AWS.

## Lessons Learned

- Importance of least-privilege access
- Difference between users, groups, and policies
- How IAM improves AWS security
- Managing permissions safely
